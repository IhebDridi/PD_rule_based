# Theme switching (dev vs main)

## Files

| Path | Purpose |
|------|---------|
| `_static/themes/midnight-teal.css` | Midnight Teal dark UI |
| `_static/themes/classic.css` | Empty stub (original look) |
| `templates/global/Page.html` | Loads theme CSS when enabled |
| `settings.py` → `SESSION_CONFIG_DEFAULTS['use_midnight_teal']` | Master switch |

Style guide (local notes): `midnight-teal-style-guide-for-otree.txt`

## How to turn ON (dev-branch default)

```text
OTREE_THEME=midnight-teal
```

Or leave unset — default on this branch is `midnight-teal`.

Restart the oTree server after changing the env var.

## How to turn OFF (for main / production)

Either:

1. Set environment variable before starting oTree:

```text
OTREE_THEME=classic
```

2. Or in `settings.py`, change the default:

```python
use_midnight_teal=environ.get("OTREE_THEME", "classic") == "midnight-teal",
```

When merging to **main**, prefer default `"classic"` so production stays on the old look without requiring env vars.

## Per-session override

Session configs inherit `use_midnight_teal` from `SESSION_CONFIG_DEFAULTS`. You can override one config:

```python
dict(
    name='...',
    ...,
    use_midnight_teal=False,
)
```

## Rollback guarantee

All Midnight Teal styling lives in `_static/themes/midnight-teal.css` and loads **only** when
`session.config.use_midnight_teal` is true (from `OTREE_THEME=midnight-teal` or settings default).

With `OTREE_THEME=classic`:
- `midnight-teal.css` is **not** loaded
- `classic.css` is loaded (empty — no overrides)
- Page templates keep their original inline `<style>` blocks unchanged

No HTML template edits are required to switch themes.

## Note

Page templates ship light inline `<style>` blocks. Midnight Teal overrides them with
`!important` rules scoped under `html.otree-theme-midnight-teal` (class added in `Page.html`
when the theme is active).
